CPU?
-> 컴퓨터에서 일어나는 대부분의 계산과 명령 처리를 담당하는 핵심 부품

하는 일?
-> 연산, 제어, 데이터 처리

내부 구성?
-> 산술논리장치, 제어장치, 레지스터, 캐시 등이 포함된 여러개의 코어로 구성

각각의 역할
- 산술논리장치(ALU) : 연산 수행
- 제어장치(Control Unit) : 명령 해석 및 실행 제어
- 레지스터(Register) : 연산에 필요한 데이터를 가장 빠르게 저장 및 사용
- 캐시(Cache) : RAM보다 빠르게 접근할 수 있는 고속 메모리

Fetch–Decode–Execute 사이클이란?
-> CPU가 메모리에서 명령을 가져오고, 해석하고, 실행하는 기본 동작 반복 구조

각각의 역할
- Fetch (가져오기) : 메모리(RAM)에서 다음에 실행할 명령어를 읽어옴
- Decode (해석하기) : 가져온 명령어가 무슨 의미인지 해석
- Execute (실행하기) : 실제로 연산 수행

병목 현상?
-> CPU가 프레임 시간 내에 로직 처리를 완료하지 못해 전체 성능이 제한되는 현상

원인?
-> 작업량이 많거나 작업 방식이 비효율적일 경우 발생

해결 방안?
-> 연산량 감소, 알고리즘 개선, 메모리 관리 최적화, 멀티스레딩 적용 등의 방법을 통해 해결 가능
